Algorithm Book Updated (1) : New Specification

2024/12/02 TlogyVine Label: C++ struct

1、 Pay attention to 'stack overflow', main is extremely fragile! (Warning)

The main can only hold about 4MB of data, and an int array with a length of 1E6 can easily explode it Especially in the case of "engineering problems", it allows you to perfect a structure. Note that it is important to check whether the instance of the structure is actually running on the stack!

How can you look at it? If the official provides the code for instantiation of the structure, you can determine it based on its location. As long as it is not initialized at a global location, not dynamically applied, or the outer class is not dynamically applied, it is considered to be running on the stack. What if the official code for instantiation is not provided? Then 'assume' that it runs on the stack, and write code based on this premise thereafter.

If it runs on the stack, avoid allocating too much static space within the structure! For instance, the aforementioned 1E6_int array, if included, will definitely cause a stack overflow. Even if a single instance doesn't overflow, if multiple instances are instantiated, it will still overflow.

So, what's our solution? I recommend using STL containers like vectors, which allocate memory on the heap. You may have heard that continuous push_down in vectors can reduce performance and resist using them. My opinion on this is: so foolish! you solve problems narrowly, so why bother with performance? Remember, premature optimization is the root of all evils!
